# Player Management System

**Module:** CSE 4002 – Fundamentals in Programming  
**Qualification:** HD in Computing and Software Engineering / Network Technology and Cyber Security

---

## 📖 Project Overview

The **Player Management System** is a console-based C++ application developed for the **Upcountry Warriors Baseball Club**. The system automates the process of managing player records, replacing the club's manual record-keeping system.

It provides a user-friendly menu-driven interface for managing players and teams while storing data permanently using file handling techniques.

---

## 🎯 Objectives

- Automate player information management.
- Reduce manual record-keeping.
- Store player data permanently using files.
- Provide an easy-to-use console interface.
- Demonstrate structured and modular programming concepts in C++.

---

- 🔐 User Login

## 📖 User Guide

1. Launch the application.
2. Log in using your username and password.
3. Choose an option from the main menu.
4. Perform required operations such as:
   - Add Player
   - Display Players
   - Search Player
   - Manage Teams
   - View Team Details
5. Logout or Exit when finished.

---

## 💾 Data Storage

The application stores player information using files.

Stored information includes:

- Player Registration Number
- First Name
- Last Name
- Date of Birth
- Runs Scored
- Team Information

Data remains available after the application is closed.

---

## ⚠ Assumptions

- Every player has a unique registration number.
- A player can belong to a maximum of two teams.
- Login credentials are required to access the system.
- Input validation prevents invalid entries.
